# 智慧城市融合架构设计

在实现智慧城市功能的所有环节中，相关数据构成不同性质的信息发挥了不可替代的作用。随着智慧城市信息共享能力的提高，供决策分析的数据量将不断增长。同时，接受来自不同渠道的数据能力，也要求对数据的一致性、准确性、权威性、即时性和完整性给予确认。智慧城市数据融合架构促进了数据生成、交互、挖掘与融合，是智慧城市建设发展的核心支撑。

智慧城市数据融合是对位于不同地域的信息形成、管理单位中分散存储和管理各类信息资源，通过一定的手段使之连接成为一个结构有序、管理一体、配置合理的整体，将数据融合在一个平台，实现用数据说话、用数据治理、用数据方面民生。具体而已，就是依托数据共享交换平台，建设统一的大数据中心，创新信息采集和挖掘分析模式，建立基础数据的集中采集、多方利用工作机制，实现跨部门、跨层级的业务协同，通过深化拓展大数据融合应用，实现信息资源的充分利用与最大增值。

数据融合架构设计思路遵循如下几条原则：

  1. 加强数据统筹，破除信息孤岛：把政府部门的共享、公开和开放数据（如企业库、人口库、地理信息库等基础数据库）全部集中统筹，不在各个部门系统里横向交互，各部门需要的共享、基础数据直接跟数据统筹资源来交互。
  2. 实现从点对点交换到一对多共享：打破传统“点到点”数据交换共享模式，更加注重可共享、基础的数据落地，创新一对多的书记共享新模式，实现所有部门在遇到跨部门、跨网络数据需求时，能够通过数据中心进行共享，破解政务信息孤岛、业务互联互通难、业务协同难等方面的问题。
  3. 促进由业务驱动到数据驱动的升级：通过数据加工、数据整合、数据关联登功能，动态配置实现各类不同主题的信息处理，在人口库、法人库、基础数据库的基础上，通过清洗、转换、集成，构建业务应用所需的业务数据库，提升数据的价值，实现数据向信息的转变。面向领导管理决策需求，对所有智慧城市管理业务数据进行汇总、统计，实现基于智慧城市运行监测分析模型的高端管理应用。通过分析模型的建立，将这些数据有机关联起来并进行分析挖掘，有效预测城市未来发展，为城市规划提供强大的决策支持。
  4. 加强集约建设，推进大数据应用：加强政府部门在管理和服务过程中对数据的主动采集，推进无线识别技术、传感器、无线网络等新技术的广泛应用，提高数据采集的智能化水平，打破地区和部门数据壁垒，实现数据资源整合共建、广泛共享，依托部门和区域间的信息共享和业务协同，建立政府和社会互动的大数据采集机制，制定政府数据共享开放目录，将共享的数据向城市各部门及社会开放，由政务数据开放推动社会数据开放。
